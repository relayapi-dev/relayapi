## Trigger Model

Every automation has exactly one `trigger`. When an event arrives that matches the trigger type + its config + its filters, the trigger matcher creates an enrollment and queues the graph to run.

A trigger spec has four parts:

```json
{
  "type": "instagram_comment",
  "account_id": "acc_abc123",
  "config": { "keywords": ["info", "link"], "match_mode": "contains", "post_id": null },
  "filters": { "tags_none": ["blocked"], "segment_id": "seg_vip" }
}
```

- **type** — one of the values listed below.
- **account_id** — scopes the trigger to a specific social account (optional for virtual triggers like `scheduled_time`).
- **config** — trigger-type-specific matching (keyword list, post scope, cron, threshold, ref URL slug).
- **filters** — contact-side gates: tag includes/excludes, segment membership, custom-field predicates.

## Tier 1 — Full Conversational (DM + comment + reply)

These platforms support both inbound DMs and public engagement events.

### Instagram

| Type | Fires on |
| --- | --- |
| `instagram_dm` | Direct message received |
| `instagram_comment` | Comment on a post or reel |
| `instagram_story_reply` | User replies to one of your stories |
| `instagram_story_mention` | User mentions your account in a story |
| `instagram_mention` | User @-mentions your account in a post/comment |
| `instagram_reaction` | Message reaction (emoji) added/removed |

Note: Instagram does not expose a follower webhook through the public Graph API v25, so Follow-to-DM flows must be enrolled manually.

## Cross-Platform

Triggers that are not tied to a single platform:

| Type | Fires on |
| --- | --- |
| `scheduled_time` | Cron schedule — daily digests, weekly check-ins |
| `engagement_threshold` | A post hits N likes / N comments / N shares (see `engagement_rules` config) |
| `tag_applied` | Contact has a tag added |
| `tag_removed` | Contact has a tag removed |
| `field_changed` | Contact custom field changes value |
| `external_api` | External system posts to `POST /v1/automations/:id/enroll` |
| `manual` | Only enrolled manually — used for internal flows + template scaffolding |
| `segment_entered` | Contact becomes a member of a segment |
| `segment_left` | Contact leaves a segment |

## Filters

All triggers support the same filter schema:

```json
{
  "tags_any": ["vip", "trial"],
  "tags_all": ["newsletter_opted_in"],
  "tags_none": ["blocked", "unsubscribed"],
  "segment_id": "seg_enterprise",
  "predicates": {
    "all": [
      { "field": "fields.country", "op": "eq", "value": "IT" },
      { "field": "fields.lifetime_value", "op": "gte", "value": 100 }
    ]
  }
}
```

Filter predicates resolve against contact columns, custom fields (`fields.<slug>`), tags, and enrollment state captured earlier in a run.
