## Overview

Engagement rules let you auto-amplify posts based on real-time performance metrics. Define a trigger metric and threshold, and the system will check your post at regular intervals — when the threshold is met, the configured action fires automatically.

This is useful for programmatic amplification: retweet your own post once it hits 100 likes, or auto-reply with a promo link when a post takes off.

## How It Works

1. **Create a rule** with a trigger metric, threshold, action type, and check schedule
2. The system checks the post's metrics at each interval
3. When the metric meets or exceeds the threshold, the action is executed
4. If the threshold is never met within the check window, the rule expires with no action taken

```bash
# Create an engagement rule
curl -X POST https://api.relayapi.dev/v1/engagement-rules \
  -H "Authorization: Bearer rlay_live_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Auto-retweet popular posts",
    "account_id": "acc_xxx",
    "trigger_metric": "likes",
    "trigger_threshold": 100,
    "action_type": "repost",
    "check_interval_minutes": 360,
    "max_checks": 3
  }'
```

## Supported Platforms and Actions

Not all actions are available on every platform. The table below shows what is supported:

| Action | Description | Platforms |
|--------|-------------|-----------|
| **Repost** | Retweet/reshare the original post | Twitter, LinkedIn, Bluesky |
| **Reply** | Post a reply to the original post | Twitter, LinkedIn, Threads, Bluesky, Facebook |
| **Repost from account** | Repost from a different connected account | Twitter, LinkedIn |

For the **reply** action, you must also provide `action_content` — the text to post as a reply.

For the **repost from account** action, you must provide `action_target_account_id` — the connected account that will perform the repost.

## Check Schedule

The check schedule is controlled by two fields:

| Field | Description | Default |
|-------|-------------|---------|
| `check_interval_minutes` | Time between each metric check | 360 (6 hours) |
| `max_checks` | Maximum number of checks before the rule expires | 3 |

The total observation window is `check_interval_minutes * max_checks`. For example, with the defaults (360 min interval, 3 checks), the system checks the post at **6 hours**, **12 hours**, and **18 hours** after publication.

If the threshold is met on any check, the action fires immediately and the rule is marked `triggered`. If no check meets the threshold, the rule expires as `completed` with no action taken.

## Trigger Metrics

| Metric | Description |
|--------|-------------|
| `likes` | Total like/favorite count |
| `reposts` | Total repost/retweet/reshare count |
| `replies` | Total reply/comment count |
| `impressions` | Total impression/view count |

Metric availability depends on the platform and what it reports via its API.

## Rule Statuses

| Status | Description |
|--------|-------------|
| `active` | Rule is active and checks are scheduled |
| `triggered` | Threshold was met and the action was executed |
| `completed` | All checks exhausted without meeting the threshold |
| `paused` | Rule is paused (can be reactivated) |
| `error` | An error occurred during execution |

## SDK Usage

```typescript
import Relay from "@relayapi/sdk";

const client = new Relay({ apiKey: "rlay_live_..." });

// Create a rule
const rule = await client.engagementRules.create({
  name: "Auto-retweet popular posts",
  account_id: "acc_xxx",
  trigger_metric: "likes",
  trigger_threshold: 100,
  action_type: "repost",
  check_interval_minutes: 360,
  max_checks: 3,
});

// List rules
const { data } = await client.engagementRules.list({ limit: 20 });

// Get a specific rule
const r = await client.engagementRules.get("er_abc123");

// Update a rule
await client.engagementRules.update("er_abc123", {
  trigger_threshold: 200,
});

// Delete a rule
await client.engagementRules.delete("er_abc123");
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/v1/engagement-rules` | Create a rule |
| `GET` | `/v1/engagement-rules` | List rules (paginated) |
| `GET` | `/v1/engagement-rules/:id` | Get rule details |
| `PATCH` | `/v1/engagement-rules/:id` | Update a rule |
| `DELETE` | `/v1/engagement-rules/:id` | Delete a rule |
